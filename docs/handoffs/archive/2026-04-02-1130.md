### Architecture decisions

- **Single source of truth:** `src/config/site.ts` — all HAWC-specific values. Primary city is McKinney (GHL/SEO target), legal address is Plano.
- **Brand injection:** `Layout.astro` uses `define:vars` → CSS custom properties → consumed in global.css/Tailwind. Orange `#ec5b13` = `--brand-primary`. Dark brown `#221610` = `--brand-secondary`. Note: components use the actual hex values directly (not the CSS vars) because Tailwind v4 JIT can't scan dynamic var references.
- **Contact flow:** Form POST → `/api/contact` (Supabase insert) → 303 redirect to `/contact/success` → async fire-and-forget `/api/contact-sync` (GHL upsert + opportunity + note). Contact form never fails due to GHL downtime.
- **Content placeholders:** All 15 service pages and 9 location pages use `CONTENT_PLACEHOLDER` in `src/data/services.ts` and `src/data/locations.ts`. Rankability script replaces these after first deploy.
- **No local git:** All file management via `mcp__plugin_github_github__push_files`. `/tmp/hawc-build/` was the on-disk source but no clean remote configured.
- **GHL IDs hardcoded in site.ts (not secrets):** `GHL_LOCATION_ID`, `GHL_PIPELINE_ID`, `GHL_STAGE_ID` are in `src/config/site.ts` as non-secret config. Only `GHL_API_KEY` is a secret (env var only).

### Design system

- **Fonts:** Plus Jakarta Sans (display) + DM Sans (body) — self-hosted from `/public/fonts/`
- **Brand colors:** Primary `#ec5b13` (orange CTA), Secondary `#221610` (dark nav), Accent gold `#e2a03f` (stars/highlights on dark), Navy `#2c3e50` (section backgrounds), Terracotta `#b05d45` (darker orange for text/icons on light bg)
- **WCAG verified:** All text/bg combos annotated with contrast ratios in component comments. AA compliant throughout.
- **Buttons:** `.btn-primary` (orange fill), `.btn-secondary` (navy outline), `.btn-navy` (navy fill), `.btn-ghost` (white/translucent on dark), `.btn-submit` (full-width orange)
- **Cards:** `.card-base` (white, border, rounded-2xl, hover shadow), `.card-photo` (4:3 ratio photo container), `.card-body` (padded content)
- **Scroll reveal:** All sections use `.reveal-item` + `IntersectionObserver` pattern. No GSAP on production pages (vanilla JS only for scroll reveal).

### Key invariants (never violate)

- `SUPABASE_SERVICE_ROLE_KEY` — server-side only, never `PUBLIC_`
- `GHL_API_KEY` — server-side only, never `PUBLIC_`
- `RANKABILITY_API_KEY` — server-side only, never `PUBLIC_`
- WCAG AA: 4.5:1 normal text, 3:1 large/UI — all annotated in component comments
- Max 2 fonts. Max animation scale 1.03.
- Contact form writes to Supabase first. GHL sync is async fire-and-forget — never blocks the user redirect.
- `src/config/site.ts` is the only place HAWC-specific values live. Never hardcode business data in components.
