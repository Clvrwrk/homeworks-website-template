## Decisions Made This Session

- **`created_by` is TEXT not UUID** — WorkOS user IDs are strings like `user_01ABCDEF`, not UUIDs. Deviation from spec noted in migration SQL.
- **`neighborhood` AND `community` columns kept separate** — existing code uses `neighborhood` for filter chips (projects/index.astro) and `community` for display on detail page (projects/[slug].astro). Both hold the same subdivision name but serve different UI roles.
- **Migration 003 in Plan 3 (not Plan 1)** — Services/Locations tables added mid-session; kept in Plan 3 to avoid rewriting Plan 1. Plan 3 Task 10 runs this migration before Agency Admin routes.
- **Grandfathered projects skip publish gate** — 6 existing hardcoded projects seeded as `status='published'` with empty `before_photo_url`. Publish gate (requires both photos + all fields) applies to new records only.
- **No React in admin** — All admin interactivity (readiness bar, scope items, local links, photo preview) is vanilla JS in `<script>` tags, matching existing codebase pattern. The prior handoff said "React islands" — that decision was reversed. No React exists in this codebase.
- **`createAdminClient()` uses service role key** — bypasses RLS for all admin write operations. All admin API routes must validate WorkOS session before using it.
- **GHL notify is fire-and-forget** — never throws, never blocks the admin UI. Silently fails if GHL is down.
- **`seo-documents` storage bucket** — new bucket needed for Services/Locations SEO doc uploads (in addition to `project-photos`).
- **`PUBLIC_GOOGLE_MAPS_KEY` env var needed** — for Events hub and Event detail page Google Maps embeds.

### Architecture decisions
- **Astro SSR (not static)** — required for Supabase queries and WorkOS auth. `output: "server"` with `@astrojs/node` standalone adapter.
- **No React** — All interactivity is vanilla JS `<script>` tags. The brainstorm mentioned React islands; this was reversed after reading the actual codebase. GSAP + vanilla JS is the pattern.
- **Three Supabase client patterns:**
  - `createServerClient()` — anon key, respects RLS, used in public pages
  - `createAdminClient()` — service role key, bypasses RLS, server-side only
  - `getBrowserClient()` — singleton, anon key, browser only
- **WorkOS session:** cookie `hw_admin_session`, 7-day expiry, httpOnly, signed with `WORKOS_COOKIE_PASSWORD`
- **Three roles: `agency_admin`, `client_admin`, `worker`** — Agency manages all; Client manages their team; Workers draft-only (Publish hidden entirely)
- **Publish gate is server-side** — enforced in API route, not just client readiness bar
- **OG fetch runs server-side only** — press mention URL parsing never touches browser
- **GHL fire-and-forget** — `ghlNotifyPublish()` never throws, never awaited in critical path

### Design system
- Font: Syne (headings) + DM Mono (data) + DM Sans (body)
